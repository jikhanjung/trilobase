### 1c. Agnostida Order

0.1.8에서는 Agnostida가 Trilobita 하위 Order로 default profile에 포함.
0.2.0에서는 `adrain_2011.txt` 소스가 "Agnostida is excluded from Trilobita sensu stricto"로 명시하여 default profile에서 제외.
→ treatise1959, treatise2004 프로필에서는 정상 포함.

## 2. Treatise 1959 Profile 차이

| 항목 | 건수 |
|------|-----:|
| 0.1.8 edges | 1,768 |
| 0.2.0 edges | 1,645 |
| Common (동일 child+parent) | 150 |
| Moved (같은 child, 다른 parent) | 1,473 |
| Lost (0.1.8에만 존재) | 145 |
| New (0.2.0에만 존재) | 22 |

**차이 원인:**

1. **ALL CAPS 정규화 (주요 원인)**: 0.1.8에서 Treatise 소스의 `CYCLOPAGNOSTIDAE` 등 ALL CAPS 이름이 canonical DB의 `Cyclopagnostidae`와 별도 taxon으로 생성됨 → 0.2.0에서 title case 정규화로 동일 taxon과 매칭 → taxon ID가 달라져 edge가 "moved"로 분류
2. **Fuzzy matching 제거**: 0.1.8은 JSON 기반 fuzzy matching으로 taxon 매칭, 0.2.0은 직접 파싱으로 정확 매칭
3. **Lost 145건**: ALL CAPS 정규화로 중복이 합쳐진 결과 (28 families, 84 genera, 22 subfamilies 등)

**Lost families 28건** (모두 0.2.0 taxon 테이블에는 존재):

Treatise 1959에서 ALL CAPS로 기재된 family/subfamily가 0.1.8에서는 별도 ID로 생성됐으나 0.2.0에서는 canonical DB의 기존 taxon과 합쳐짐. Edge가 다른 ID로 기록되므로 "lost"로 보이지만 실제로는 중복 해소.

예: `CYCLOPAGNOSTIDAE` (id=5369, 0.1.8 신규) → `Cyclopagnostidae` (id=67, canonical, 0.2.0에서 재사용)

**Moved 1,473건**: 대부분 위와 같은 ID 변경으로 인한 parent 변경. 실질적 분류 변경은 아님.

**주의**: 일부 "moved" 항목 중 family가 genus 아래로 배치되는 경우 발견 (예: Corynexochidae → Wenkchemnia). Treatise 1959 소스 파일의 들여쓰기 파싱 이슈로 추정. 추후 검토 필요.

## 5. 요약

| 구분 | 판정 | 설명 |
|------|------|------|
| Default profile | ✅ 개선 | 97.7% 동일, 나머지는 데이터 품질 개선 |
| Treatise profiles | ⚠ 주의 | ALL CAPS 정규화로 edge 수 감소 (정상), 일부 파싱 이슈 잔존 |
| SYNONYM_OF | ✅ 개선 | +20건 추가 (canonical opinions 임포트) |
| 신규 PLACED_IN | ✅ 개선 | +30건 (canonical parent_id 폴백) |
| Agnostida | ℹ 의도적 | Adrain 2011 원문 반영 (treatise 프로필에서는 포함) |

### 추후 검토 사항

1. `Family?` 이름 (Dalmanitidae?, Nileidae? 등 9건) — questionable marker 처리 통일
2. Treatise 1959 일부 "moved" 항목에서 family→genus 역전 현상 — 들여쓰기 파싱 검증
3. `Leny` family 컨테이너 (4 genera) — 불완전 데이터로 인한 잔존, 원본 데이터 수정 필요
