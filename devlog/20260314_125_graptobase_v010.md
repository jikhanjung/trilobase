# Graptobase v0.1.0 — Graptolite Taxonomy DB 빌드

**날짜:** 2026-03-14
**작업 번호:** 125

## 요약

Treatise on Invertebrate Paleontology Part V (Graptolithina)의 3개 에디션에서 추출한
소스 텍스트 파일(`data/sources/treatise_graptolite_*.txt`)을 기반으로 graptobase DB 및
.scoda 패키지를 생성하는 빌드 스크립트를 구현했다.

brachiobase 빌드 스크립트를 기본 템플릿으로 사용하되, graptolite 소스 데이터의 특수한
분류 구조를 처리하기 위해 파서를 확장했다.

## 생성된 스크립트

### `scripts/build_graptobase_db.py`
- 3개 프로파일 빌드 (1955, 1970, 2023)
- `Subclass`, `Infraorder` 랭크 추가 지원
- graptolite 소스 전용 파서 확장 (아래 상세)
- `temporal_code_mya` 테이블 생성 (PaleoCore + CAM/MSIL/LCARB 수동 추가)
- Timeline 쿼리 6개 포함 (geologic + pubyear 축)
- Timeline axis를 데이터 보유 시대만 필터 (MCAM~MISS, 10 steps)

### `scripts/build_graptobase_scoda.py`
- brachiobase scoda 빌드 스크립트를 graptobase용으로 적응

## Graptolite 전용 파서 확장

### 추가 랭크
- `Subclass` (2023 에디션: Graptolithina를 Subclass로 취급)
- `Infraorder` (2023: Diplograptina, Neograptina)

### 스킵 패턴 (비공식 섹션 헤더)
- `Section GONIOGRAPTI` 등 Section 헤더
- `Group GRAPTOVERMIDA` 등 Group 헤더
- `Stem group NEOGRAPTINA` 등 Stem group
- `Addendum` 이후 전체 (1970 중복 기재 방지)
- `Nomen dubium`, `Taxa previously identified as land plants`
- `Multiramous forms`, `Taxa with cladial branching`
- `Linograptinae with cladia`, `Streptograptines`

### Incertae sedis 헤더 처리
- `Graptolithina incertae sedis` → 스택을 Graptolithina로 리셋
- `Dendroidea incertae sedis` → 스택을 Dendroidea로 리셋
- `Normalograptidae incertae sedis` → 스택을 해당 Family로 리셋
- 패턴: `_INCERTAE_SEDIS_HDR_RE = r'^(\w+)\s+incertae\s+sedis$'`

### Unrecognizable Genera
- `Unrecognizable Genera` 헤더 감지 → 스택을 root로 리셋
- 하위 genera의 `is_valid = 0` 설정

### Type species / Temporal code 추출
- `[*type_species]` 를 파서에서 직접 추출 (bracket 제거 전)
- `| | TEMPORAL_CODE` 에서 temporal_code 추출
- `?` prefix 제거 (e.g., `?MCAM-LCARB` → `MCAM-LCARB`)

### Root 브리징
- Profile 1/2: Class Graptolithina로 orphan roots 연결
- Profile 3: Subclass GRAPTOLITHINA → Class Graptolithina 브리지 (1건)

## 함께 수정된 사항

### Brachiobase + Graptobase: Detail view 수정
- `type: "table"` / `type: "breadcrumb"` → scoda-engine이 인식 불가
- `type: "linked_table"` + `sub_queries` 패턴으로 변환 (trilobase 방식)
- `type: "field_grid"` 명시, Parent 필드에 클릭 링크 추가

### scoda-engine: Timeline 빈 데이터 무한 루프 수정
- **원인:** `morphToStep()`에서 빈 트리 return 시 `currentIdx` 미갱신
  → `playTimeline()` while 루프가 같은 index 무한 반복
- **수정:** `static/js/app.js`
  - `morphToStep()`: base/compare root가 null이면 `currentIdx = toIdx` 후 return
  - `loadStep()`: root null일 때 캔버스 clear 추가

### Temporal code 보완
- `MSIL` (Middle Silurian, 433.4-427.4 Mya) 추가
- `CAM` (Cambrian, 538.8-485.4 Mya) 추가
- `LCARB` (Lower Carboniferous, 358.9-323.2 Mya) 추가

## UI Manifest 특징

- **Taxonomy Tree**: Family leaf rank, genera count, temporal_code 컬럼 포함
- **Tree Chart**: Subclass/Infraorder 포함 rank_radius 설정
- **Profile Comparison**: Diff Table + Diff Tree + Side-by-Side + Animation
- **Timeline**: Geologic (MCAM~MISS, 10 steps) + Publication Year 축
- **Detail View**: sub_queries 기반 linked_table (trilobase 패턴)
